Directed Graph

A directed graph (digraph) $G=(V,E)$ consists of a non-empty set $V$ of vertices (nodes) and a set $E\subseteq V\times V$ of directed edes which are ordered pairs of vertices.
This allows self-loops $(v,v)\in E$ where $v\in V$ but not for parallel edges. We can also have anti-parallel edges $(u,v)(v,u)\in E$ where $u,v \in V.$

We use notation $u\rightarrow v$ for both the edge $(u,v)$ itself and for the fact that $(u,v)\in E$. $u\rightarrow v$ is an outgoing edge from $u$ and an incoming edge to $v$.
If $u\rightarrow v$ then $v$ is a successor of $u$ and $u$ is a predecessor of $v$. Moreover, $u$ and $v$ are neighbors, when $u\rightarrow v$ or $v\rightarrow u$, a symmetric relationship. Since we allow self-loops, a vertex can be its own neighbor in which case it is also its own predecessor and its own successor.
Isolated vertices: vertices with no neighbors.
Edgeless digraph: just like an edgeless graph.

Digraph Terminology

Degrees

The outdegree of a vertex $u$ is the number of successors of $u$, same as the number of outgoing edges from $u$, denoted out($u$).
Similarly, the indegree of a vertex $u$ is the number of predecessors of $u$, same as the number of incoming edges to $u$, denoted in($u$).
The degree of a node is deg($u$) $=$ out($u$) + in($u$). A node of indegree 0 is called a source and a node of outdegree 0 is called a sink.
Proposition: The sum of the outdegrees of all vertices equals the sum of the indegrees of all vertices and further equals the number of edges. Therefore, the sum of the degrees equals twice the number of edges.

Directed Walk and Path

A directed walk is a non-empty sequence $u_0,u_1,\ldots,u_k$ such that $u_0\rightarrow u_1 \rightarrow \ldots \rightarrow u_k$.
We call this a directed walk from $u_0$ to $u_k$ of length $k$.
A directed path is a walk with no repeated vertices.
For every vertex $v$ there is a directed path of length 0: $v$.
Do not confuse the directed walk of length 0, $u$ with the directed walk of length 1 given by the existence of a self-loop: $v\rightarrow v$.
Walks of length 0 are paths but walks of length 1 given by self-loops are not paths.

Directed Cycles

A directed cycle is a closed walk $u_0\rightarrow u_1 \rightarrow \ldots \rightarrow u_k \rightarrow u_0$, with $u_0,u_1,\ldots,u_k$ all distinct. The length of the cycle is $k + 1$.
There are no cycles of length 0. A self-loop gives a cycle of length 1. A cycle of length 2 consists of two vertices and edges between them in opposite directions (antiparallel edges).
Path digraphs, cycle digraphs (including of length 1 and 2) and digraph isomorphism are defined similarly to the undirected case.

Reachability

A vertex $v$ is reachable from a vertex $u$ when there is a walk (and therefore a path) from $u$ to $v$. We write $u\twoheadrightarrow v$ for the reachability relation.
Proposition: The reachability relation is reflexive, i.e., $u\twoheadrightarrow u$ and transitive, i.e., $u\twoheadrightarrow v$ and $v\twoheadrightarrow w$ imply $u\twoheadrightarrow w$.
The relation $u\twoheadrightarrow v$ is called the reflexive-transitive closure of the (edge) relation $u\rightarrow v$.

Strong Connectivity

Two vertices of a digraph, $u$ and $v$, are strongly connected when $v$ is reachable from $u$ and $u$ is reachable from $v$, i.e., $u\twoheadrightarrow v$ and $v\twoheadrightarrow u$.
Proposition: Strong connectivity is reflexive, symmetric and transitive.
A set of vertices is strongly connected when any two of its vertices are strongly connected.
The maximally strongly connected sets of vertices are called strongly connected components (scc’s).
Proposition: Any two distinct strongly connected components are disjoint.
Corollary: The strongly connected components determine a partition of the vertices (but not of the edges!).

Topological Sort

A topological sort of a digraph is a sequence $\sigma$ in which every vertex appears exactly once (i.e., a permutation of its vertices) such that for any edge $u\rightarrow v$ in the graph, the vertex $u$ appears in $\sigma$ before (but not necessarily immediately before) the vertex $v$.
Proposition: If a digraph has a topological sort then:

  • The first vertex in the sort is a source and the last is a sink.
  • The digraph is a DAG.

Proposition: Every DAG has at least one source and at least one sink.
Proposition: Every DAG has at least one topological sort.

Topological Sort Construction Algorithm

Input: a DAG $G = (V,E)$ with $n$ vertices.
Output: a permutation $\sigma$ of $V$ that is a topological sort.

  1. $G$ has at least one source, $u$. Remove $u$ and call $G_u$ the resulting digraph.
    (Removing $u$ cannot create a cycle since we only delete edges. $G_u$ is also a DAG.)
  2. Recursively construct a topological sort of $G_u$, i.e., a permutation $\sigma$ of $V \backslash \{u\}$. Then, concatenate $u$ at the beginning.
  3. Output $u\sigma$ is a topological sort of G.

Binary Trees

Rooted Trees

A rooted tree is a pair $(T,r)$ where $T = (V,E)$ is a tree and the vertex $r \in V$ is designated as a root.
Proposition: Any edge of a rooted tree is traversed in the same direction by all unique paths from the root to each of the other vertices.
Using this direction, make the rooted tree into a digraph, in fact, a DAG with the root as source and tree leaves (or the root, when alone) as sinks.

Rooted Tree Terminology

We call leaves the nodes of degree 1 in rooted trees. However, it is convenient to state that the single node of a one-node rooted tree is also a leaf, although it has degree 0.

root: $r$.
leaves: $1,5,8,6$.
child $=$ successor
parent $=$ predecessor
height $=$ distance from root to farthest leaf height $=$ 4.
binary(rooted) tree: every node has at most 2 children.

Binary Trees

Proposition: A binary tree of height $h$ has a maximum of $2^{h+1} − 1$ nodes among which are $2^h$ leaves. This maximum is attained for the complete binary tree of height $h$.
We define a complete binary tree of height $h$ to be a rooted tree in which every non-leaf node has two children and all leaves are at distance $h$ from the root.

Adjacency Matrices

Two vertices in a graph are adjacent if there is one edge that is incident to both of them.
An adjacency matrix for a digraph has one row and one column associated with each vertex. If there is an edge from the vertex of the row to the vertex of the column, then the value in that position is 1. Otherwise, it is 0.

We say an adjacency matrix because the order of the vertices is arbitrary!
For an undirected graph, we can define an adjacency matrix three ways:

  1. Only use the part strictly above the diagonal.
  2. Only use the part strictly below the diagonal.
  3. Copy each value to both locations (symmetric matrix).

Incidence Matrices

An edge is incident to a vertex if that vertex is either the beginning or the end of that edge.
An incidence matrix for a digraph has one row associated with each edge and one column associated with each vertex. If an edge runs from vertex $a$ to vertex $b$, the row corresponding to that edge has -1 in column $a$ and 1 in column $b$; all other entries in that row are 0.
We say an incidence matrix because the order of the edges and the order of the vertices are arbitrary!
Unlike adjacency matrices, incidence matrices are often not square! In fact, a square incidence matrix indicates that the graph has at least one cycle.

The Connected Components Basis

Definition: Let $V$ be a set of vectors. A set of vectors $B$ is called a basis for $V$ if

  1. the vectors in $B$ are linearly independent and
  2. Span($B$) $=$ $V$.

Proposition: Let $G$ be a graph and $A$ be its incidence matrix. A basis for the set of vectors $\vec x$ such that $A \vec x = \vec0$ represents a partition of $G$ into its connected components. The full set of vectors is called the nullspace of $A$.

The Cycle Basis

Proposition: Let $G$ be a graph and $A$ be its incidence matrix. A basis for the set of vectors $\vec x$ such that $A \vec {x}^T = \vec0$ represents a reduced set of the undirected cycles in $G$. The full set of vectors is called the left nullspace of A.
